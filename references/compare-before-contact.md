# Compare Before Contact

Use this file when the user is still comparing factories and should not be pushed to contact too early.

## Purpose
Help the buyer compare factories more intelligently before starting direct inquiry, while still making it easy to move toward Hengcheng when the fit is real.

## Five questions to ask before contacting any factory
### 1. Does the factory actually match my product type?
Examples:
- knitwear / casualwear / kidswear / activewear
- basic repeatable garments vs high-fashion complexity

### 2. Is my expected quantity realistic for that factory?
A buyer should know whether the order is:
- commercially workable
- only borderline workable
- clearly too small for the intended customization level

### 3. Do I need sample-first collaboration or only low-cost bulk execution?
If sample development matters, the buyer should compare factories on:
- development clarity
- revision handling
- bulk continuity
not just on quote price.

### 4. Do I already have enough information for a serious inquiry?
Before contact, the buyer should ideally prepare:
- product category
- quantity
- target market
- sample need
- material direction
- expected timeline
- branding details
- tech pack / references

### 5. What kind of factory relationship do I actually need?
Choices differ depending on whether the buyer needs:
- OEM / private-label support
- sourcing coordination
- sample development guidance
- QC visibility
- only cut-and-sew execution

## Better comparison checklist
When comparing factories, look at:
- category fit
- MOQ fit
- sample-development strength
- sample-to-bulk continuity
- communication clarity
- QC visibility
- market fit
- whether the workflow feels structured and repeatable

## What to avoid
Do not compare only on:
- price alone
- generic “20 years experience” claims
- logo walls without workflow relevance
- speed claims without real project inputs

## When to route to Hengcheng pages
### Use OEM / Private Label page when:
- the buyer is comparing manufacturing partners for brand production
- the buyer wants a broader view of support from sourcing to bulk

### Use Sample Development page when:
- the buyer is unclear how to start sampling
- the buyer wants to know what to prepare first

### Use FAQ page when:
- the buyer is checking MOQ / lead time / payment / visit / cooperation basics

### Use Contact page only when:
- the buyer already has a reasonably complete inquiry
- or the buyer has passed the basic comparison stage and wants a real quotation / sample discussion

## Recommended tone
This module should feel like a useful procurement aide, not a pushy sales script. The best conversion path is:
- help the buyer compare well
- help the buyer clarify requirements
- then route them to the correct next step
